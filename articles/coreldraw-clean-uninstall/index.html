CorelDRAW 彻底卸载指南

ISSUE: newer version is already installed

卸载 CorelDRAW 后安装旧版时提示”已安装更新版本”？本文讲解如何彻底清理残留。

为什么会这样？

Windows 判断软件是否安装过，不只看 Program Files 目录，还会检查：

  1. 卸载注册表项：“程序和功能”背后的登记信息
  2. Windows Installer 数据库：MSI 安装包的”户口本”
  3. Installer 缓存文件：通常在 C:\Windows\Installer
  4. 用户配置和缓存：AppData、ProgramData
  5. 外围痕迹：兼容性记录、许可服务等

所以你看到的情况是：

  • 程序文件没了
  • 桌面图标没了
  • 但安装器的”户口本”还在
  • 旧版安装器检测到记录，直接拦截

这是 Windows Installer + Corel 安装链残留 共同导致的。

残留藏在哪里？

第一层：文件夹残留

  • C:\Program Files\Corel
  • C:\Program Files (x86)\Corel
  • C:\ProgramData\Corel
  • C:\Users\用户名\AppData\Roaming\Corel
  • C:\Users\用户名\AppData\Local\Temp\Corel

第二层：用户配置残留

  • HKCU\Software\Corel\...

第三层：MSI/Installer 残留（最关键）

  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\{GUID}
  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Installer\UserData\...\Products\...
  • HKCR\Installer\Products\...
  • C:\Windows\Installer\*.msi

第四层：外围痕迹

  • ShellCompatibility\Applications\coreldrw.exe
  • Protexis Licensing Service 相关记录

清理顺序

  1. 正常卸载 CorelDRAW 及相关组件
  2. 清理文件夹残留
  3. 清理用户注册表残留
  4. 处理 MSI/Installer 残留
  5. 清理外围痕迹
  6. 重启后安装目标版本

不要把第 4 步提前，Installer 层最危险也最关键

第一步：备份

创建备份目录：C:\Users\用户名\CorelCleanupBackup

备份以下注册表项（导出 .reg 或用 reg query 保存 .txt）：

  • HKCU\Software\Corel\...
  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\{GUID}
  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Installer\UserData\...\Products\...
  • HKCR\Installer\Products\...

删除以下目录（如果存在）：

第三步：清理用户配置

删除注册表项：

包括所有子项（如 CorelDRAW\22.0、25.0、26.0 等）。

第四步：处理 MSI/Installer 残留

这是最关键的一步。

4.1 检查卸载项

位置：HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall\{GUID}

查找 InstallSource 指向 Corel 安装目录的项。

4.2 检查 Installer UserData

位置：HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Installer\UserData\S-1-5-18\Products\...

检查：

  • InstallProperties
  • LocalPackage
  • InstallSource

4.3 检查 HKCR\Installer\Products

位置：HKCR\Installer\Products\...

检查：

  • ProductName
  • PackageName
  • SourceList
  • LastUsedSource

4.4 处理 MSI 缓存

不要直接去 C:\Windows\Installer 删文件！

正确做法：

  1. 通过 LocalPackage 确认 MSI 文件路径
  2. 确认该 MSI 确实绑定 Corel
  3. 先尝试标准卸载
  4. 无效再手动删除

第五步：清理外围痕迹

ShellCompatibility 记录

Protexis Licensing Service 事件源

Installer\Folders 目录索引

查找指向 Corel 目录的值并删除。

第六步：验证清理

确认以下 5 件事：

  1. 文件夹没了：ProgramData、AppData 中的 Corel 目录
  2. 用户配置没了：HKCU\Software\Corel
  3. 卸载项没了：InstallSource 指向 Corel 的 GUID 项
  4. Installer 记录没了：UserData 和 HKCR 中的 Corel 相关项
  5. 重启后安装旧版不再报错

简化版流程

  1. appwiz.cpl 正常卸载 CorelDRAW
  2. 重启
  3. 删除文件夹残留
  4. 删除 HKCU\Software\Corel
  5. 检查并清理 MSI/Installer 残留
  6. 清理外围痕迹
  7. 重启
  8. 安装目标版本

常见误区

误区正确做法
只删 Program FilesInstaller 数据库才是主因
只看”程序和功能”MSI 记录可能不在列表中
看到 VBA 就认为和 Corel 无关检查 InstallSource 是否指向 Corel
直接清 C:\Windows\Installer只能删确认属于 Corel 的 MSI
删完不重启就安装重启让安装器状态刷新
